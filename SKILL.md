# 克制情感叙事文写作

> 本 skill 面向"抖音故事号"/"微信对话故事"赛道，核心风格是：**让读者自己哭，自己心疼，自己脑补，作者一个字都不说。**

## 第一步：理解这个文类的本质

### 它是什么

一种以"微信对话截图感"为载体，用克制到极点的笔法呈现禁忌情感（体制内隐忍、同性暗流、师生微妙、婚外克制）的非虚构叙事短文。

通常 1500-3000 字，结构是：
- 开场：深夜朋友圈/消息触发
- 发展：白天正常上班，深夜消息往来
- 转折：某个具体事件（车里对话/停车场遇见/调动）
- 结尾：不告别、不议论、不升华，只有一个小动作或一句短消息

### 它不是什么

- ❌ 不是"我的心好痛"的情绪宣泄文
- ❌ 不是"他这样做是因为他爱我"的议论文
- ❌ 不是"后来我想通了"的鸡汤总结文
- ❌ 不是微信截图的简单堆砌，需要叙事衔接

### 爆款共性（底层逻辑）

这类文章之所以能病毒传播，底层是以下几个情绪钩子：

1. **夜情境霾**——深夜+独处+手机屏幕，是现代人最脆弱的场景
2. **说不得**——禁忌情感（体制内权力差/同性/婚外）说出口就失去，必须憋着才动人
3. **不对等**——他发了消息，她纠结了一整天才回，他却已经睡了
4. **小事堆出深情**——"今天食堂有你爱吃的糖醋排骨"比"我爱你"更戳人
5. **一个留到最后才说的词**——整篇不说"爱"或"喜欢"，只在结尾出现一次，或者永远不说

## 第二步：素材输入与分析

用户提供素材时，按以下维度分析，不全也没关系：

| 维度 | 说明 | 重要性 |
|------|------|--------|
| 人物关系 | 体制内上下级/师生/同事/异性/同性 | 核心 |
| 禁忌类型 | 权力差、同性、婚外、单向暗恋 | 核心 |
| 触发事件 | 深夜朋友圈、加班、应酬、调动 | 高 |
| 对话样本 | 有具体对话最好，没有就描述 | 高 |
| 核心意象 | 月亮、晚安、窗外、糖醋排骨 | 中 |
| 结尾走向 | 告别/调动/沉默/错过 | 高 |

## 第三步：写作规则（铁律）

### 铁律1：禁用所有情绪形容词

禁止出现以下词汇及同义词：

| 禁用 | 替用 |
|------|------|
| 紧张/忐忑/心慌 | 用身体动作：手心出汗、手抖、反复看手机 |
| 感动/心动/温暖 | 用沉默/动作：他没有说话/递了杯水 |
| 难过/伤心/痛苦 | 用日常崩塌：吃不下饭/忘了喝水/走神 |
| 喜欢/爱 | 永远不直接说，用行为和对话暗示 |
| 害怕/恐惧 | 用回避：绕路/换时间/找借口 |
| 纠结/矛盾 | 用延宕：打了又删/回了一半 |

**正确示例：**
- ❌ "他发来消息，我很紧张，心跳加速"
- ✅ "消息弹出来，我盯着屏幕看了三秒，才想起要回"

### 铁律2：作者不许出场议论

全文不允许出现"我想""我认为""我觉得他知道"等主观判断句。读者的理解必须完全来自对话和动作。

允许的唯一例外：结尾最后一句可以有感，但不超过15字，且不能有"我"字。

**正确示例：**
- ❌ "那一刻我知道，他是喜欢我的"
- ✅ "他把车窗摇上去，开走了。尾灯消失在路口。"
- ✅ 结尾允许："有些话，到嘴边，又咽回去了。"

### 铁律3：对话要短，要残缺

对话不是戏剧台词，是微信消息。微信消息的特征：
- 不完整（"你也这么晚？"没主语）
- 可以只有一个字（"嗯"）
- 可以没有声音（对方发来一首歌，没有文字）
- 可以有timestamp的暗示（"00:23 刚发"）

**正确示例：**
- 他：「你也这么晚？」
- 我：「嗯」
- 他：「我是说……算了，早点睡」
- 我：「好」
- （他发来一张照片，窗外夜景，没有配文）

### 铁律4：场景代替情绪

这是"Show Don't Tell"的核心技术。

| 要表达的情绪 | 禁用写法 | 正确场景写法 |
|------------|--------|------------|
| 紧张 | "我很紧张" | "我盯着对话框，手指悬在键盘上，没动" |
| 克制 | "他在忍耐" | "他端起酒杯，喝了一口，放下的时候比端起来的时候轻" |
| 深情 | "他很爱我" | "他把我们部门的人都记住了，包括我爱吃什么" |
| 告别 | "我们要分开了" | "他把东西装进纸箱，在我工位前停了三秒，没说话" |

### 铁律5：至少埋两个意象，结尾必须回调

**意象池（可用，也欢迎创造新的）：**

```
场景类：月亮/夜色/路灯/车窗/办公室窗/停车场/雨/空调/食堂
物品类：一件外套/雨伞/旧本子/一张照片/一条语音/成绩单
行为类：一条消息/晚安/微信运动步数/外卖红包
```

**意象分类：**
- **场景意象**：需要是物理场景（夜色、路灯、雨）
- **物品意象**：故事里出现的实物（本子、照片、成绩单），callback时以"物证"形式重现
- **行为意象**：某个重复出现的动作（发了又撤、存下来不删）

**Callback 结构：**
- 开头出现"月亮"→ 结尾必须有月亮的场景或消息
- 开头出现"糖醋排骨"→ 结尾需要有食物相关的告别
- 开头出现"晚安"→ 结尾必须以晚安结束
- 物品类意象的callback：结尾重现这个物品，但不是通过对话说出来，而是"她把它存了下来/又翻出来看了一眼"

**禁止：** 结尾用议论回收意象（"那轮月亮，就像他对我说的那句晚安"←禁止）

### 铁律6：对话之间的叙事要极简

对话和对话之间的叙事衔接，用句号断成独立短句，每句不超过15字。不要用"然后"连接，不要写流水账。

**正确示例：**
- "灯关了。手机屏幕还亮着。"
- "他站在车旁边，没有发动车子。"
- "雨不大。他把伞收了。"

### 铁律7：转折要用具体事件承载

不能写"后来我心态变了"这种空洞转折。转折必须是一个：
- 具体场景（停车场看见他的车/他在车里没开灯）
- 具体对话（"你知道我为什么今晚回来吗"）
- 具体动作（他没有发动车子/他提前回来没有说）

### 铁律8：言行矛盾是最强的张力

**这是此类文章最核心、也最容易被忽略的技术。**

最好的情感张力不在"他说了什么"，而在"他说的和他做的完全相反"：

- 他说"以后不要联系了"，但他还在发消息
- 他说"发错了"，但他发的是她最想听的那句话
- 他说"公事公办"，但他的关心比公事多了一句"带伞"

**写法要点：**
- 先建立他的"言行一致"（他说的和做的看起来都对）
- 然后埋入矛盾（他说要保持距离，但他的行为暴露了相反的东西）
- 不要点破，让读者自己发现
- 读者发现的那一刻，比任何情绪形容词都更有力

**禁止：** 点破矛盾（"他嘴上说不要联系，却一直在发消息"←这是议论，不是场景）

## 第四步：结构模板

### 基础结构（适用于大多数素材）

```
【第一幕：触发】（300-400字）
- 具体时间/地点/场景开头（不要写"有一天"）
- 深夜朋友圈/消息作为入口
- 领导/对方发来第一条消息
- 第一轮对话

【第二幕：拉扯】（800-1000字）
- 白天正常上班，对话变少但出现细节关心
- "今天有雨带伞""食堂有你爱吃的"
- 某个白天的小互动（办公室门口/食堂/走廊）
- 深夜消息频率增加
- **关键：建立言行矛盾。** 他说公事，但关心比公事多；他表面冷淡，但细节暴露了在意。读者发现矛盾的那一刻，比任何情绪词都更有力。

【第三幕：转折】（400-500字）
- 具体事件：车里对话/他在等我/调动通知
- 核心对话（全文最重要的一句，通常很短）
- 留白

【尾声：告别/错过】（200-300字）
- 不说再见，不说原因
- 最后一个动作或最后一句消息
- 意象callback（呼应开头）
- 结尾不许议论（少数允许15字感）
```

## 第五步：正文写作（逐句指导）

### 开头写法

**推荐开场方式：**

1. **消息触发型：** 深夜，发来一条没有前缀的消息
2. **朋友圈触发型：** 加班到很晚，发了一条朋友圈，什么都没写
3. **时间地点型：** "那年冬天，办公室的暖气坏了"

**绝对禁止的开头：**
- "大家好，今天给大家讲一个故事"（严禁）
- "在我二十多岁的时候"（太老套）
- "我有一个朋友"（一眼假）
- "随着时代的发展"（完全禁止）

### 对话写法

每条消息单独成段，用「」包裹，格式：
```
他：「消息内容」
我：「消息内容」
```

消息之间的叙事夹叙，每段不超过10字，用句号断句。

### 叙事衔接词库

不用"然后"，用以下词自然衔接：
- 灭了（灯灭了）
- 划过去（把消息列表往上划）
- 打了（打了几个字，删了）
- 没回（这条消息，我没有回）
- 放下（把手机放下了）
- 锁屏
- 刷了刷（朋友圈）
- 停了一下（动作突然中断）

### 结尾写法

结尾有四种可选：

1. **消息戛然而止型：** 最后一条消息发出，对方没有再回，故事结束
2. **意象留白型：** 以一个场景描写收尾，不解释
3. **动作收尾型：** 以一个人的动作为结尾，他没有说话
4. **回调意象型：** 开头出现的意象在结尾再次出现，但不解释

**坚决禁止的结尾：**
- 总结升华型（"这个故事告诉我们……"）
- 议论感慨型（"有些感情，注定是没有结果的……"）
- 鸡汤收尾型（"愿所有深情都不被辜负"）
- 注明来源型（"根据读者倾诉撰写"←原文有这个，但AI写不要加）

## 第六步：四层质检（写完必跑）

### ⚠️ 写前检查：逻辑一致性（先于L1）

**动笔之前，先把人物状态线写清楚。**

写之前必须确认以下三条线：

**① 主线生死状态**（人物最终走向）
- 明确：主角/关键人物最终是活着/死了/分开了
- 如果涉及多次事件（如两次手术），确保每次生死状态变化有清晰交代
- 死人说的话/做的事，必须是闪回、遗言、或生前记录的，不能是死后还出现的行为

**② 禁忌关系逻辑自洽**
- 权力差禁忌：两个人的身份差是否始终一致？（领导→永远不能变成平级）
- 情感边界：暧昧可以，但两人之间的互动必须符合现实逻辑（不能越级、不能突然表白）
- 言行一致：他说的和做的之间要有矛盾，而不是说的和做的一致（说的一致=没有张力）

**③ 时间线无穿越**
- 每个事件标注清楚时间顺序
- 避免"之后有一天他突然……"这种没有时间锚点的跳跃
- 换季/节日/节假日可作为自然时间标记（"那年过年"/"那年三月"）

**典型逻辑错误（立即修复）：**
- 死人做手术（"二次手术"后发现前面已经死了）❌
- 事件顺序颠倒（A发生在B之后但A被写在前面）❌
- 身份突然变化（领导突然变同事/同事突然变领导）❌
- 行为和身份不符（病人给护士送餐但护士在ICU值夜班）❌

### L1 硬性规则（零容忍）

全文搜索以下词汇，有则必改：

**情绪形容词禁用：**
紧张、忐忑、心慌、心跳、紧张、感动、温暖、难过、伤心、痛苦、喜欢、爱（直接说）、害怕、纠结、矛盾、心酸、心疼、委屈、失落、孤独、寂寞

**议论句式禁用：**
我想、我觉得、我认为、我知道、我明白、我意识到、他知道、他明白、他觉得
（角色在对话里的"我知道""我觉得"不计入，仅限旁白议论）

**禁止开头词：**
首先、随着、大家好、在当今、有一天（平铺直叙的）、我的朋友、今天我要讲

**禁止结尾词：**
这个故事告诉我们、愿、愿你、愿所有、所以我们、其实（总结性的）

**错别字高发区（写完必查）：**
- 的/地/得 用错（场景描写用"地"，动词前用"得"）
- "一起"vs"提前"（"忘了跟你说"不要写成"忘了一起跟你说"）
- 了/着 混用（"收了起来"不要只写"收"）
- 掉/了 混用（"死掉了"不要写成"死了"再出现新对话）

### L2 情感克制检查

- 全文有没有超过5个字的情绪形容词？（5字以内可接受）
- 有没有作者直接判断"他这样做是因为……"的句子？
- **言行矛盾：** 有没有"他嘴上说X，行动却暴露了Y"的结构？让读者自己发现矛盾，不要作者点破
- 对话中的情感有没有超过暧昧的边界？（暧昧可以，表白不可以）
- 结尾有没有超过15个字的议论/感慨？

### L3 逻辑一致性检查

**① 生死状态链**
- 关键人物最终是活着/死了/分开了？状态是什么？
- 每一个生死状态变化点，文中有没有交代清楚？（没有交代=逻辑漏洞）
- 死人的行为：死人有没有在死后还做动作？（"二次手术"之前必须确认人还活着）

**② 人物行为一致性**
- 他的身份在全文是否始终一致？（领导→全程是领导，不能中途变同事）
- 他的说话方式是否一致？（从头到尾叫"温护士"→突然叫"你"→是否有过渡？）
- 他的行为是否和他的处境一致？（一个在ICU门口守夜的人→有没有时间和精力去做某事？）

**③ 时间线一致性**
- 每个事件的时间顺序是否符合逻辑？（用"那年过年""那年三月"等季节锚点检查）
- 有没有"先发生的事写在后面，后发生的事写在前面"的情况？

**④ 逻辑自洽（读者视角）**
- 读到这里，有没有一个时刻你觉得"这里不对"？（那就是要修复的）
- 有没有一件事，没有任何铺垫就发生了？（没有前因的后果=逻辑断层）

### L4 意象Callback与言行矛盾检查

- 开头埋了哪些意象？结尾回调了几个？
- 是否有至少一个意象在结尾再次出现但不解释？
- 结尾的意象出现，是动作/场景，还是议论？
- 全文有没有言行矛盾的张力点？（检查：他说的话 vs 他做的事，有没有形成反差）

### L5 错别字扫描

全文扫一遍，重点检查：
- 的/地/得 是否用对
- "一起"vs"提前"（"忘了跟你说"不要写成"忘了一起跟你说"）
- "收起来"vs"收了起来"（"了起来"比"收"更完整）
- 句子里有没有多字/少字（朗读一遍，听起来不对的地方重点查）

### L6 阅读体验检查

以读者视角快速读一遍：
- 有没有哪个地方你会想"这段跳过去"？（那就是要删的）
- 对话之间的叙事衔接够不够自然？
- 全文最高潮的那句话够不够短？（越短越有力）
- 读完有没有"想说点什么又说不出来"的感觉？（这是正确效果）

## 附录：典型素材模板

用户提供素材时，按以下格式整理（不用全有，越核心的越要有）：

```
【关系】领导/老师/同事/男同/异性/婚外（说清楚权力位）
【禁忌类型】权力差暗恋/单向/双向暗涌/错过
【核心场景】深夜消息/车里对话/调动
【对话1】第一条深夜消息（原文或描述）
【对话2】最具代表性的一句（短/越界）
【意象1】已经存在的：月亮/夜景/糖醋排骨/外套
【意象2】可以新造的：任何物品或动作
【结尾】希望走向：告别/调动/沉默/再无交集

【生死状态】关键人物最终是活着/死了/分开了（必填）
【逻辑风险】有没有可能造成前后矛盾的点？（收到素材后先检查，有矛盾要向用户确认）
```

**收到素材后，必须先做逻辑一致性检查再动笔：**
- 确认生死状态线：主角和关键人物的最终结局是什么？中间有没有生死变化？
- 确认身份关系线：两人在整个故事里身份是否始终一致？
- 如果素材里有矛盾（如"他说了一句话但前面他已经死了"），先向用户确认，不能假设。
